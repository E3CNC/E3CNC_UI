## Architecture Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Release distribution | GitHub release with `e3cnc-stack-v*.tar.zst` | Self-contained, checksummed, CI-built |
| Nginx routing (single instance) | `server_name e3cnc.local` on port 80 | Coexists with other web services on same machine |
| Nginx routing (multi-instance) | Hostname-based (`printer1.local`, etc.) | Clean URLs, single port, extends single-instance |
| Frontend deployment | Pre-built in release artifact, also supports `docker cp` from host's `dist/` for testing | Release-based for prod, local build for dev |
| Vendor code location | All modifications merged INTO `vendor/` snapshots | Release artifact is self-contained; no separate sync step |
| CLI architecture | `cli/` package with focused modules | 1,896→1,006 lines, single responsibility per file |
| Ansible roles | bootstrap-stack + extractor + moonraker-config + macros | Agent and klipper-extras now redundant (vendor-included) |
